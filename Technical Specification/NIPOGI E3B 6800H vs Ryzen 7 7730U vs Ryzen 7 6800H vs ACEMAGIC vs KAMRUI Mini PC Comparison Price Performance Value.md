## 2) GPU / vidéo / “qualité lecture” : là, le 6800H met une claque

- **6800H** : iGPU **Radeon 680M (RDNA2, 12 CUs)** + **AV1 decode** + plateforme plus moderne. ([Notebookcheck](https://www.notebookcheck.net/AMD-Ryzen-7-6800H-Processor-Benchmarks-and-Specs.589871.0.html))

### Codec AV1 (YouTube/streaming moderne)

- **Radeon 680M** : **AV1 decode matériel** (moins de CPU, moins de chauffe, moins de bruit). ([CPU Monkey](https://www.cpu-monkey.com/en/igpu-amd_radeon_680m?utm_source=chatgpt.com))
- **Vega 8** : **pas d’AV1 matériel**. ([CPU Monkey](https://www.cpu-monkey.com/en/igpu-amd_radeon_rx_vega_8_raven_ridge?utm_source=chatgpt.com))

**Conclusion vidéo :**

- Si tu regardes beaucoup de streaming 4K (YouTube en AV1), le **6800H/680M** est objectivement plus “clean” (CPU moins sollicité, ventilation plus calme) que les puces Vega. ([Notebookcheck](https://www.notebookcheck.net/AMD-Ryzen-7-6800H-Processor-Benchmarks-and-Specs.589871.0.html))

> Audio : en HDMI/DP, c’est **numérique**, donc la “qualité” ne varie quasiment pas entre mini-PC. La vraie différence, c’est plutôt **silence** (ventilo) et stabilité (drivers, chauffe), pas le son.

## 3) RAM + SSD : pourquoi ça explose (ou pas) le prix

### DDR4 16 Go vs DDR5 32 Go

- **32 Go DDR5** (souvent sur les configs 6800H) = plus cher **et** plus utile si :
  - gros multitâche (IDE + Docker + VM + navigateur 40 tabs),
  - iGPU sollicité (la RAM = VRAM),
  - besoin de “future-proof”.
- **16 Go DDR4** (souvent sur 7730U/V2748) : OK bureautique/dev “classique”, mais tu sens vite la limite si tu fais du lourd.

### 512 Go vs 1 To SSD

- 1 To = surcoût direct + souvent meilleure endurance (selon modèle).
- Attention au piège : certains mini-PC annoncent “M.2 SSD” mais c’est parfois **SATA** au lieu de **NVMe** (énorme différence en I/O).

Le 7730U est une plateforme plus “ancienne” (DDR4/LPDDR4x, PCIe 3, Vega). ([Notebookcheck](https://www.notebookcheck.net/AMD-Ryzen-7-7730U-Processor-Benchmarks-and-Specs.681481.0.html))

## 5) Refroidissement / bruit / conso : le “coût caché” du 6800H

- **6800H (45W)** : plus performant, mais **plus dur à refroidir** dans un petit boîtier → potentiellement **plus de bruit** et parfois **throttling** si le design est cheap. ([cpubenchmark.net](https://www.cpubenchmark.net/cpu.php?cpu=AMD+Ryzen+7+6800H&id=4749))
- **7730U (15W)** : souvent **silencieux**, très bon rendement perf/watt. ([cpubenchmark.net](https://www.cpubenchmark.net/cpu.php?cpu=AMD+Ryzen+7+7730U&id=5215))
- **V2748 (embedded, TDP haut)** : peut être très bien si le châssis est pensé pour tenir la charge, mais sur les mini-PC “grand public”, ça dépend énormément du refroidissement. ([cpubenchmark.net](https://www.cpubenchmark.net/cpu.php?cpu=AMD+Ryzen+Embedded+V2748&id=4319))

## 6) “Sur quoi se base le prix ?” (la mécanique simple)

En pratique, le prix d’un mini-PC monte surtout à cause de :

1. **CPU + plateforme**

- 6800H = segment plus “haut” + iGPU beaucoup plus fort + features modernes → coûte plus cher. ([Notebookcheck](https://www.notebookcheck.net/AMD-Ryzen-7-6800H-Processor-Benchmarks-and-Specs.589871.0.html))

1. **RAM (quantité + génération)**

- 32 Go DDR5 > 16 Go DDR4 (coût matière + marge).

1. **SSD (capacité + type + qualité)**

- 1 To NVMe correct > 512 Go SATA basique.

1. **Refroidissement + VRM + carte mère**

- un bon dissipateur/ventilo/VRM = coût réel, impact direct sur perf soutenue et bruit.

1. **I/O “premium”**

- 2.5GbE, Wi-Fi 6E, USB4, etc. (si réellement présent). ([Notebookcheck](https://www.notebookcheck.net/AMD-Ryzen-7-6800H-Processor-Benchmarks-and-Specs.589871.0.html))

1. **Licence Windows** (théorique) + SAV/marge

- parfois flou sur certaines marketplaces : “Win11 Pro” n’explique pas tout, mais ça peut jouer.

Donc :

- **7730U** = souvent **plus silencieux**, moins chaud, meilleur en “PC salon / bureautique H24”.
- **6800H** = peut être **plus bruyant** si le châssis/cooling est moyen… MAIS même bridé, il reste généralement devant.

> Point important : sur mini-PC, **le vrai boss s’appelle le “power limit + refroidissement”**. Un 6800H mal refroidi peut throttler. Mais à performances égales, il part de plus haut.

## 5) RAM / SSD : la partie où les fiches produit mentent parfois 😄

### RAM (le piège n°1)

- Sur beaucoup de configs “32 Go”, tu peux avoir **1×32 Go** (single channel).
- Pour un APU (surtout la 680M), **le dual channel change tout** (bande passante mémoire).

**À vérifier avant achat / à réception :**

- idéal : **2×16 Go**
- si tu reçois **1×32**, prévois d’ajouter une barrette pour passer en dual-channel.

### SSD (le piège n°2)

“512 Go M.2” peut être :

- un NVMe correct,
- ou un NVMe très moyen,
- ou parfois même du SATA M.2 (selon modèle).

Pour du dev + VM + gros projets, un SSD médiocre se sent (latences/IOPS).

## 2) La vraie différence : **16 Go vs 32 Go**, et “dual-channel ou pas”

### Pourquoi 32 Go change tout

- **Dev / Docker / VM / IDE + navigateur** : 16 Go → tu arrives vite au swap → machine “lente” même si le CPU est monstrueux.
- **iGPU Radeon 680M** (sur 6800H) : la RAM sert aussi de “VRAM”.
  En **single-channel**, tu brides les perfs graphiques (et parfois la fluidité multi-écrans/4K dans certains cas).

**Donc** :

- si le modèle à **298€** arrive en **1×16 Go**, il est “moins cher” mais tu n’exploites pas le 6800H à fond tant que tu n’as pas mis une 2ᵉ barrette.
- le modèle à **388€** a *plus de chances* d’être déjà “prêt à performer” (souvent 2×16), mais ce n’est pas garanti : certains vendeurs livrent aussi 1×32.

------

## 3) Rapport qualité/prix (calcul froid)

Comme le CPU est le même, le “value” brut CPU/€ favorise le moins cher :

- **298€** : tu payes le 6800H au prix cassé → excellent **si** tu assumes l’upgrade RAM.
- **388€** : tu payes surtout **+16 Go de RAM** et la tranquillité “plug & play”.

La question devient : **combien te coûte de passer de 16 à 32 Go toi-même ?**

- Si le mini-PC a **2 slots SODIMM accessibles** : tu ajoutes une barrette et tu récupères **dual-channel + 32 Go**.
- Si c’est soudé/1 slot/galère SAV : là le 388€ devient rationnel.
